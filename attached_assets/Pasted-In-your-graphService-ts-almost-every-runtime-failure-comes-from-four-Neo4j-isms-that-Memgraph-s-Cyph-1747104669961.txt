Minimal patch set

Below is a surgical diff (line numbers approximate) that keeps your fallback logic intact while making every query that reaches Memgraph run cleanly:

@@ createNode
-const query = `CREATE (n:${node.type} $props) RETURN n`;
+const query = `
+  CREATE (n:${node.type})
+  SET n = $props          // sets every key from the parameter map
+  RETURN n
+`;

@@ createEdge
-        CREATE (source)-[r:${relationship} $props]->(target)
-        RETURN source, target, type(r) as relationship
+        CREATE (source)-[r:${relationship} $props]->(target)
+        RETURN r, source, target, type(r) AS relationship

@@ getAllNodes
-RETURN n ORDER BY n.id SKIP toInteger($skip) LIMIT toInteger($limit)
+RETURN n ORDER BY n.id SKIP $skip LIMIT $limit

@@ getAllEdges
-          RETURN type(r) AS relationship, 
+          RETURN type(r) AS relationship,

@@ executeCustomQuery (fallback vector-search sniff)
-if (query.includes('db.index.vector.queryNodes')
+if (query.includes('vector_search.search')

❯ **NEW helper (optional, but handy)**
   export const VECTOR_QUERY = `
     CALL vector_search.search($index, $limit, $embedding)
     YIELD node, similarity
     WITH node, similarity
     WHERE similarity >= $minSim          // client-side filter
     RETURN node, similarity
   `;

A couple of bonus clean-ups

Drop toInteger() everywhere. It does nothing for Memgraph and can break prepared‐statement caching.
Prefer WITH collect() over double queries for the sub-graph method; it cuts the round-trip in half.
Edge strength default: move the 0.7 constant into a DEFAULT_EDGE_STRENGTH so you only patch in one place next time.
