Vector Similarity in MAGE:**

**Indexing Features:**

*   **Vector indexing capabilities for faster similarity searches:**
    *   Yes, Memgraph provides this via `CREATE VECTOR INDEX`.
    *   It uses a `READ_UNCOMMITTED` isolation level specifically for vector indices to maintain performance while ensuring ACID properties for other database operations.
*   **Best practices for index creation with embeddings:**
    *   **Mandatory Parameters:** Always specify `dimension` and `capacity`.
    *   **Capacity:** `capacity` is a minimum and prefers powers of two. It's adjusted internally.
    *   **Metric:** Choose an appropriate `metric` based on your embedding type and similarity notion (e.g., `cos` for many language model embeddings, `l2sq` is the default).
    *   **`resize_coefficient`:** Controls how the index grows when capacity is reached (default is 2).
    *   The example creates an index: `CREATE VECTOR INDEX index_name ON :Node(vector) WITH CONFIG {"dimension": 2, "capacity": 1000, "metric": "cos","resize_coefficient": 2};`
*   **Index limitations for embedding vectors:**
    *   "Properties are stored as 64-bit values in the property store and as 32-bit values in the vector index." This implies a potential precision reduction when vectors are added to the index, using `f32` by default.
    *   The query planner doesn't automatically use vector indices for WHERE clauses. You must use the `vector_search.search()` procedure.
