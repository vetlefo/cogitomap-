Overhaul Context Analyzer (# client/src/lib/ContextAnalyzer.ts):
Goal: This should no longer primarily generate keywords/importance itself from raw text. Its main job now is to take the StructuredLLMOutput from the backend and translate it into BubbleNode and Edge objects for the useVisualization store.
Modify analyzeMessage Function:
Change its signature. It should now probably accept the original Message and the StructuredLLMOutput (if available) as input, along with existingNodes and maybe the previousMessageId to link conversation flow.
Node Creation:
Create the primary BubbleNode for the main_response (type user_message or ai_message). Use the summary from the structured data if available, otherwise use the first part of main_response as content. Use sentiment or other factors to influence initial importance.
Iterate through identified_topics. For each topic, create a new BubbleNode with type: 'topic', content: topic_name. Calculate its position relative to the main message node.
Iterate through key_entities. For each entity, create a new BubbleNode with type: 'entity', content: entity.entity, and store entity.type in metadata. Position it near the main message node.
(Self-Correction: Need IDs for these new nodes. Generate unique IDs like topic-${topic_name}-${timestamp} or entity-${entity.entity}-${timestamp}).
Edge Creation:
Create a primary Edge linking the previousMessageId to the new main message node's ID (relationship response_to).
Create edges from the main message node to each topic node (relationship: 'mentions').
Create edges from the main message node to each entity node (relationship: 'mentions').
(Future enhancement: Use internal_links from the structured output if the LLM provides them for more specific relationships).
